# 从零学习路线

## 第 1 阶段：先理解问题

我们想做的不是“让模型背下所有资料”，而是让系统在回答前先去资料库里找证据。

最小目标：

```text
用户提问 -> 找到相关文档片段 -> 展示答案和来源
```

这就是 RAG 的核心思想。

## 第 2 阶段：理解 Transformer 在哪里

Transformer 在本项目里主要负责“理解文本含义”，具体做法是把文本变成向量。

比如：

```text
"注意力机制会给重要词更高权重" -> [0.12, -0.03, 0.88, ...]
```

向量不是给人看的，它是给机器计算相似度用的。

## 第 3 阶段：理解文档切块

真实文档通常很长，不能整篇直接塞给模型，所以要切成小片段。

切块时要考虑：

- 片段太短：上下文不够，意思容易断。
- 片段太长：检索不精确，也可能超过模型输入长度。
- 需要 overlap：相邻片段保留一点重叠，避免关键句子刚好被切断。

## 第 4 阶段：理解向量检索

把每个文档片段变成向量之后，用户的问题也会变成向量。然后系统计算：

```text
问题向量 和 每个片段向量 的相似度
```

相似度越高，说明这个片段越可能回答问题。

## 第 5 阶段：升级方向

掌握最小版本后，可以继续加：

- PDF 解析：支持上传论文和资料。
- FAISS 或 Chroma：支持更多文档和更快检索。
- reranker：让另一个 Transformer 再精排检索结果。
- 生成模型：把检索片段组织成自然语言答案。
- Web UI：让项目变成真正可演示的软件。
