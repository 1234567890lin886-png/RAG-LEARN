# 项目过程记录

## 第 0 步：确定项目方向

项目方向是“基于 Transformer 的个人知识库检索问答系统”。

为什么选它：

- 比单纯文本分类更接近真实应用。
- 能覆盖 Transformer、embedding、向量检索、RAG。
- 可以从命令行小工具逐步升级到完整 Web 应用。

## 第 1 步：搭建骨架

先创建清晰的目录：

- `sample_docs/` 放示例资料。
- `src/scholarmind/` 放代码。
- `docs/` 放学习笔记。

这样做的原因是：新手项目最怕所有代码挤在一个文件里，后面很难扩展。

## 第 2 步：设计最小闭环

最小闭环是：

```text
读取文档 -> 切块 -> 向量化 -> 检索 -> 输出答案和来源
```

只要这条链路跑通，后面加 PDF、数据库、网页界面都只是增强。

## 第 3 步：处理模型缓存位置

用户要求不要把模型下载到 C 盘，所以项目加入了 `--cache-dir` 参数。

这样做的原因是：Transformer 模型通常不是几 KB 的普通依赖，而是几十 MB 到几十 GB 的权重文件。实际项目里必须清楚模型下载到哪里、缓存在哪里、线上部署时怎么加载。

当前策略：

- 默认教学演示使用 `hashing`，不会下载模型。
- Transformer 后端如果没有 `--cache-dir`，不会下载到默认缓存。
- 真正运行 Transformer 时，推荐指定类似 `D:\hf-cache` 的目录。
