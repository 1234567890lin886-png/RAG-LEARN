# RAG 简介

RAG 是 Retrieval-Augmented Generation，中文常说检索增强生成。它的基本思想是：回答问题之前，先从外部知识库检索相关资料。

RAG 通常包含三个步骤。第一步是把文档切成 chunk，并用 embedding 模型向量化。第二步是根据用户问题检索最相关的 chunk。第三步是把检索结果交给生成模型组织答案。

RAG 的优点是可以使用私有资料、降低胡编概率、让答案带来源。它的局限是检索质量会直接影响回答质量，如果相关资料没有被找出来，后面的生成模型也很难答好。

一个好的 RAG 系统通常需要合适的切块策略、稳定的 embedding 模型、向量数据库、reranker，以及清晰的引用展示。
